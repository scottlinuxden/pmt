Engineering Change Proposal (ECP)
User's Manual

System Requirements
Any operating system which can execute either Netscape 4.0+ or Internet Explorer 4.0+.  JavaScript must be enabled within your browser which is the default.  Cookies should be enabled as well.  An Internet connection, dial-up or dedicated and access to the Institute for Scientific Research, Projects And Research Center website.  This web will transfer information via an SSL web server, therefore your web browser must support secure server connections through certificates.

Login Instructions
Before you can execute any ECP functions you must first login.  To login you must enter a valid username and password combination.  A username and password may be acquired by submitting e-mail to support@isrparc.org.

To login to ECP you must select Change Proposals from the Site Menu bar in the left frame of your browser.  A login screen will appear in the right frame of your web browser.  To login enter your username and password that have been provided to you and depress the Query button.  You may also depress the Help button to access this user's manual.

Change Proposal Listing
Once you have logged in you will be presented with a listing of all Change Proposals in the ECP database.  You may at this point select either Edit, Delete, or View.  Only those project members with the authority to delete will have the Delete button available for selection.

Configuring the Change Proposal Listing
The Change Proposal listing may contain a variety of information from each reported problem.  The listing is configured by selecting the field selector above each column on the listing.  To display a different Change Proposal field under the column field selector, select the drop down list on Windows platforms or the option menu selector on Unix platforms.  All possible fields that are viewable for a Change Proposal are presented for selection.  Select the field of interest and depress the Query button.  You must depress the Query button before the field data for the field selected is displayed in the Change Proposal listing.  You may depress the Query button under the "Add Column" area or at the bottom of the Change Proposal listing.

Adding Fields to the Change Proposal Listing
Additional fields may be added to the Change Proposal listing.  This is done by selecting the "Add Column" option drop down on Windows platforms or the option popup menu on Unix platforms.  Select the field you would like added to the listing and depress the Query button.  The Change Proposal listing will be re-displayed with the additional field presented on the right most part of the list.  For instance, to view only those Change Proposals which have a priority of inconvenience, select the "Add Column" drop down, select 'Priority' then depress the Query button.  The Change Proposal listing will be re-displayed will all priorities shown for each Change Proposal.  You should then select '=' then enter 'inconvenience'  in the criteria textbox, see the section Criteria Area, under the field selector, and depress the Query button.  Only those Change Proposals that have not priority of inconvenience will be displayed.

Criteria Area
The Criteria Area contains a relational operator (equal, not equal, less than, greater than, etc) drop down and a criteria textbox to provide criteria that each Change Proposal must meet before it is displayed in the Change Proposal listing.  For instance, to designate that only Change Proposals originated by 'Wise' and are priority 'inconvenience' be displayed select the equals operator under the 'originator' field selector and enter 'Wise' in the criteria textbox.  Select the equals operator under the 'priority' field selector and enter 'inconvenience' in the criteria textbox.  Do not enter the single quotes in the criteria textbox.  Note, you must currently have the 'originator' and 'priority' field selectors already displayed if they are not already by using the 'Add Column' drop down.  Select the Query button once these steps have been performed.

Creating a Change Proposal
To create a Change Proposal simply depress the Create button at the bottom of the Change Proposal Listing.  A blank Change Proposal Form will be presented for entry.  Enter all required information and depress the Save button.

Editing a Change Proposal
Editing a Change Proposal is quite simple.  Just depress the Edit button on the same row as the Change Proposal to edit.  The Change Proposal form will be displayed for editing.  Enter the information on the form that is available to you and is required, fields designated in bold and red, and depress the Save button.  Note: the data you enter is not saved until you depress the Save button.  Hitting the Back button on your web browser or leaving the form in anyway will not save the form.  Options presented on the form include Save, Delete (if authorized), View, Listing, and Help.  As mentioned earlier the Save button will save the Change Proposal, Delete will delete the Change Proposal, View will present a read-only display of the Change Proposal and is ideal for printing the Change Proposal,  Listing will display the Change Proposal Listing.  Help will present this user's manual.

Deleting a Change Proposal
Change Proposals may be deleted from the Change Proposal Listing or the Change Proposal Form.  A confirmation box will appear asking you to confirm the deletion.  If the Ok button is depressed the Change Proposal will be deleted and can not be recovered.  If you depress the Cancel button the deletion is cancelled and the Change Proposal is retained.

Viewing a Change Proposal
Viewing a Change Proposal is primarily to support viewing a read-only version of the Change Proposal.  The Change Proposal Form is displayed in a more compact manner and is presented in a an ideal formatting for printing from your web browser.

Printing a Change Proposal
Select the View button from the Change Proposal Listing or Change Proposal Form.  Select 'Print Frame' from your web browser after selecting the Change Proposal Form.

Operator Alerts
An Operator Alerts Area is presented on the Change Proposal Listing and Change Proposal Form.  This area should be monitored to verify that the operation that has been performed was executed successfully or failed.
