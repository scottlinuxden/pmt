Contact List
User's Manual

System Requirements
Any operating system which can execute either Netscape 4.0+ or Internet Explorer 4.0+.  JavaScript must be enabled within your browser which is the default.  Cookies should be enabled as well.  An Internet connection, dial-up or dedicated and access to the Institute for Software Research, Projects And Research Park website.  This web will transfer information via an SSL web server therefore your web browser must support secure server connections through certificates.

Login Instructions
Before you can execute any Contact List functions you must first login.  To login you must enter a valid username and password combination.  A username and password may be acquired by submitting e-mail to support@isrparc.org.

To login to Contact List you must select Contact List from the Site Menu bar in the left frame of your browser.  A login screen will appear in the right frame of your web browser.  To login enter your username and password that have been provided to you and depress the Query button.  You may also depress the Help button to access this user's manual.

Contact Listing
Once you have logged in you will be presented with a listing of all Project Members in the database.  You may at this point select either Edit for your contact information or View. 

Configuring the Contact Listing
The Contact Listing may contain a variety of information from each project member.  The listing is configured by selecting the field selector above each column on the listing.  To display a different contact list field under the column field selector, select the drop down list on Windows platforms or the option menu selector on Unix platforms.  All possible fields that are viewable for an Contact List are presented for selection.  Select the field of interest and depress the Query button.  You must depress the Query button before the field data for the field selected is displayed in the Contact Listing.  You may depress the Query button under the "Add Column" area or at the bottom of the Contact Listing.

Adding Fields to the Contact Listing
Additional fields may be added to the Action Item listing.  This is done by selecting the "Add Column" option drop down on Windows platforms or the option popup menu on Unix platforms.  Select the field you would like added to the listing and depress the Query button.  The Contact Listing will be re-displayed with the additional field presented on the right most part of the list. 

Criteria Area
The Criteria Area contains a relational operator (equal, not equal, less than, greater than, etc) drop down and a criteria textbox to provide criteria that each contact list must meet before it is displayed in the Contact Listing. 

Editing an Contact List
Editing an contact list is quite simple.  Just depress the Edit button on the same row as the action item to edit.  The Contact List form will be displayed for editing.  Enter the information on the form that is available to you and is required, fields designated in bold and red, and depress the Save button.  Note: the data you enter is not saved until you depress the Save button.  Hitting the Back button on your web browser or leaving the form in anyway will not save the form.  Options presented on the form include Save, View, Listing, and Help.  As mentioned earlier the Save button will save the contact data, View will present a read-only display of the contact list and is ideal for printing the contact list,  Listing will display the Contact List.  Help will present this user's manual.

Viewing a Contact
Viewing a Contact is primarily to support viewing a read-only version of the contact data associated with a project member.  The Contact List Form is displayed in a more compact manner and is presented in a an ideal formatting for printing from your web browser.

Printing Contact Data
Select the View button from the Contact List or Contact List Form.  Select 'Print Frame' from your web browser after selecting the Contact List Form.

Operator Alerts
An Operator Alerts Area is presented on the Contact Listing and Contact List Form.  This area should be monitored to verify that the operation that has been performed was executed successfully or failed.

Bug Reports
Submit all bug reports for Contact List to System Problem Report Tracking Tool or if you are not an authorized user send an e-mail.
