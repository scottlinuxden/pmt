System Problem Report (SPR)
User's Manual

System Requirements
Any operating system which can execute either Netscape 4.0+ or Internet Explorer 4.0+.  JavaScript must be enabled within your browser which is the default.  Cookies should be enabled as well.  An Internet connection, dial-up or dedicated and access to the Institute for Scientific Research, Projects And Research Center website.  This web will transfer information via an SSL web server therefore your web browser must support secure server connections through certificates.

Login Instructions
Before you can execute any SPR functions you must first login.  To login you must enter a valid username and password combination.  A username and password may be acquired by submitting e-mail to support@isrparc.org.

To login to SPR you must select Problem Reports from the Site Menu bar in the left frame of your browser.  A login screen will appear in the right frame of your web browser.  To login enter your username and password that have been provided to you and depress the Query button.  You may also depress the Help button to access this user's manual.

Problem Report Listing
Once you have logged in you will be presented with a listing of all Problem Reports in the SPR database.  You may at this point select either Edit, Delete, or View.  Only those project members with the authority to delete will have the Delete button available for selection.

Configuring the Problem Report Listing
The Problem Report listing may contain a variety of information from each reported problem.  The listing is configured by selecting the field selector above each column on the listing.  To display a different problem report field under the column field selector, select the drop down list on Windows platforms or the option menu selector on Unix platforms.  All possible fields that are viewable for a Problem Report are presented for selection.  Select the field of interest and depress the Query button.  You must depress the Query button before the field data for the field selected is displayed in the Problem Report listing.  You may depress the Query button under the "Add Column" area or at the bottom of the Problem Report listing.

Adding Fields to the Problem Report Listing
Additional fields may be added to the Problem Report listing.  This is done by selecting the "Add Column" option drop down on Windows platforms or the option popup menu on Unix platforms.  Select the field you would like added to the listing and depress the Query button.  The Problem Report listing will be re-displayed with the additional field presented on the right most part of the list.  For instance, to view only those problem reports which have a priority of inconvenience, select the "Add Column" drop down, select 'Priority' then depress the Query button.  The Problem Report listing will be re-displayed will all priorities shown for each problem report.  You should then select '=' then enter 'inconvenience'  in the criteria textbox, see the section Criteria Area, under the field selector, and depress the Query button.  Only those problem reports that have not priority of inconvenience will be displayed.

Criteria Area
The Criteria Area contains a relational operator (equal, not equal, less than, greater than, etc) drop down and a criteria textbox to provide criteria that each problem report must meet before it is displayed in the Problem Report listing.  For instance, to designate that only problem reports originated by 'Wise' and are priority 'inconvenience' be displayed select the equals operator under the 'originator' field selector and enter 'Wise' in the criteria textbox.  Select the equals operator under the 'priority' field selector and enter 'inconvenience' in the criteria textbox.  Do not enter the single quotes in the criteria textbox.  Note, you must currently have the 'originator' and 'priority' field selectors already displayed if they are not already by using the 'Add Column' drop down.  Select the Query button once these steps have been performed.

Creating a Problem Report
To create a problem report simply depress the Create button at the bottom of the Problem Report Listing.  A blank Problem Report Form will be presented for entry.  Enter all required information and depress the Save button.

Editing a Problem Report
Editing a Problem Report is quite simple.  Just depress the Edit button on the same row as the problem report to edit.  The Problem Report form will be displayed for editing.  Enter the information on the form that is available to you and is required, fields designated in bold and red, and depress the Save button.  Note: the data you enter is not saved until you depress the Save button.  Hitting the Back button on your web browser or leaving the form in anyway will not save the form.  Options presented on the form include Save, Delete (if authorized), View, Listing, and Help.  As mentioned earlier the Save button will save the problem report, Delete will delete the problem report, View will present a read-only display of the problem report and is ideal for printing the problem report,  Listing will display the Problem Report Listing.  Help will present this user's manual.

Deleting a Problem Report
Problem Reports may be deleted from the Problem Report Listing or the Problem Report Form.  A confirmation box will appear asking you to confirm the deletion.  If the Ok button is depressed the Problem Report will be deleted and can not be recovered.  If you depress the Cancel button the deletion is cancelled and the Problem Report is retained.

Viewing a Problem Report
Viewing a Problem Report is primarily to support viewing a read-only version of the problem report.  The Problem Report Form is displayed in a more compact manner and is presented in a an ideal formatting for printing from your web browser.

Printing a Problem Report
Select the View button from the Problem Report Listing or Problem Report Form.  Select 'Print Frame' from your web browser after selecting the Problem Report Form.

Operator Alerts
An Operator Alerts Area is presented on the Problem Report Listing and Problem Report Form.  This area should be monitored to verify that the operation that has been performed was executed successfully or failed.
